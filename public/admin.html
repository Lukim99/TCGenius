Recipe 편집

Bait 편집

낚시 미끼 — 각 미끼별로 보상 아이템과 가중치(rate)를 지정합니다. rate 비율에 따라 추첨됩니다.

Item 편집

아이템 목록 (배열 인덱스 = item_id). 기존 인덱스 삭제 시 모든 후속 인덱스가 변경되어 Pack/Bundle/Shop/Recipe/Bait 등과 호환되지 않을 수 있으니 주의하세요.

Equipment 편집

무기 / 갑옷 / 장신구 — 슬롯별로 인덱스가 분리되어 있습니다. 인덱스 변경 시 weapon_id / armor_id / accessory_id 참조가 깨질 수 있으니 주의하세요.

Pet 편집

펫 목록 (배열 인덱스 = pet id). 기본 구조는 장비와 동일하며, 펫 고유의 special(특수 효과)을 추가할 수 있습니다. 데이터가 아직 없어도 저장하면 새로 생성됩니다.

Fashion 편집

캐릭터 카드 스킨 — 각 스킨은 적용 가능한 캐릭터 카드 ID 목록(primary_card)과 효과 옵션(option)을 가집니다.

거래 로그

경매장/삽니다에서 체결된 거래 기록입니다. 최신순으로 표시되며 최대 2,000건까지 보관됩니다.

유생의 주사위 로그

유생의 주사위 사용 기록입니다. 최신순으로 한 줄씩 표시됩니다.

DynamoDB PITR 복원

특정 시점 데이터를 새 복원 테이블로 만든 뒤 미리보고, 문제가 없을 때 운영 테이블로 업서트 마이그레이션합니다. 운영 데이터를 삭제하지는 않습니다.

복원 작업

마이그레이션은 복원 테이블의 항목을 운영 테이블에 덮어씁니다. 운영에만 있는 항목은 삭제하지 않습니다.
